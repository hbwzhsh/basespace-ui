As we would like to make it easy for users to upload their peak files, we try our best to make the input format as relaxed as possible, with the only strict requirement that the files are tab-separated

Input format type 1: BED-like format.
Description: Any number of comments ("not starting with chrXXX") can precede the first valid line and will be ignored. Subsequently, after the first valid line, no invalid lines should be expected. Moreover, a valid line requires the 2nd and 3rd columns to be integers and 3rd column must be greater than 2nd column and not more than 10000. It would be converted to peak format by using the midpoint of the regions when running our program.
Example:
track name="testing" genome="hg18"  #all these are ignored
